- "逼近"≠"学会规则"——呼应路线图警示

- MLP 对 MNIST 分类的具体例子

### 1.3 暴露的边界
- 逼近能力 ≠ 拟合能力
- 高维空间中的几何直觉（流形假设）

## 四、训练动力学：诊断"学得怎么样" [🟡]

### 4.1 训练曲线分析
- 损失曲线的常见形态
- 欠拟合的诊断信号
- 过拟合的诊断信号
- 收敛速度与最终性能的权衡

**实操**：💻 学习曲线绘制 + 📊 4 种典型曲线对比图

### 4.2 评估方法论
- 训练集 / 验证集 / 测试集的严格隔离
- K-fold 交叉验证
- 时间序列数据的特殊处理（TimeSeriesSplit）

### 4.3 LLM 评测方法 [🔵]

#### 4.3.1 传统评估指标
- 分类：Accuracy / Precision / Recall / F1 / AUC
- 回归：MSE / MAE / R² / MAPE
- 序列任务：BLEU / ROUGE / Perplexity（见术语表）

#### 4.3.2 LLM 特有基准
- MMLU / HumanEval / TruthfulQA / GSM8K（见术语表）
- 评测维度：知识、推理、代码、事实性
- 基准选择的原则：避免单一基准过度优化

#### 4.3.3 自动化评测 vs 人类评测的权衡
- **自动化评测**：可复现、成本低、易规模化；但难以捕捉细微质量、可能存在数据污染
- **人类评测**：质量高、能捕捉细微差异；但成本高、难以规模化、标注者一致性难保证
- **LLM-as-a-Judge**：用 LLM 评测 LLM（成本/质量折中），如 GPT-4 评判其他模型
- **评测协议设计**：
  - Pairwise（两两比较）：适合 Arena 模式
  - Single-Eval（绝对评分）：适合标准化基准
  - Likert Scale（李克特量表）：适合人类标注
- **数据污染问题**：训练数据可能包含评测集，导致分数虚高

**实操**：💻 使用 lm-evaluation-harness 跑 MMLU + 📊 评测结果分析 + 🔁 评测协议完整流程

### 4.4 暴露的边界
- Goodhart 定律：指标变成目标后就不再是好指标
- 基准饱和现象（呼应第一阶段 SuperGLUE 被刷满的案例）
- 模型"答对了"与"答对了真问题"的区分

## 五、调优策略：当效果不理想时（路线图重点） [🟡]

### 5.1 偏差-方差诊断
- 高偏差（欠拟合）：模型能力不足
- 高方差（过拟合）：数据不足或正则化不足
- 学习曲线诊断法

**实操**：💻 偏差-方差权衡的可视化诊断脚本

### 5.2 数据层面的调优
- 数据增强（图像 / 文本 / 序列）
- 数据清洗与去重
- 数据平衡与采样策略
- 合成数据（Self-Play / Self-Instruct，见术语表）

### 5.3 模型层面的调优
- 正则化：L1 / L2 / Dropout / BatchNorm（见术语表）
- 模型规模选择：经验法则（**Chinchilla / Scaling Laws**，见术语表）
- 预训练 + 微调范式

### 5.4 训练层面的调优
- 学习率扫描
- Batch size 影响
- 早停（Early Stopping）
- 梯度累积（Gradient Accumulation）
- 混合精度训练（**AMP**，见术语表）

**实操**：💻 学习率扫描 + 📊 训练时间/性能权衡曲线

### 5.5 暴露的边界
- 调优的边际收益递减
- "最后一公里"问题的本质
- 何时停止调优 vs 何时重新设计
